# GitHub Repository Setup

Use this when creating the public GitHub repository.

## Repository Name

```text
ai-youtube-video-maker
```

## Repository Description

```text
Create YouTube-ready videos from a topic using AI scripts, HTML scenes, and TTS narration.
```

## GitHub CLI

After the remote repository exists, run:

```bash
gh repo edit OWNER/ai-youtube-video-maker \
  --description "Create YouTube-ready videos from a topic using AI scripts, HTML scenes, and TTS narration." \
  --add-topic ai-video-generator \
  --add-topic youtube-video-maker \
  --add-topic youtube-video-generator \
  --add-topic text-to-video \
  --add-topic script-to-video \
  --add-topic topic-to-video \
  --add-topic tts \
  --add-topic text-to-speech \
  --add-topic openai \
  --add-topic openai-api \
  --add-topic gpt-5 \
  --add-topic gpt-realtime \
  --add-topic html-video \
  --add-topic browser-video \
  --add-topic webm \
  --add-topic video-automation \
  --add-topic youtube-automation \
  --add-topic content-creation \
  --add-topic creator-tools \
  --add-topic generative-ai
```

Replace `OWNER` with the GitHub account or organization.

## Search Positioning

Use the same phrase consistently across GitHub surfaces:

- Repo name: `ai-youtube-video-maker`
- Short phrase: `AI YouTube Video Maker`
- Description: `Create YouTube-ready videos from a topic using AI scripts, HTML scenes, and TTS narration.`
- Primary keywords: `AI video generator`, `YouTube video generator`, `script to video`, `topic to video`, `TTS video`, `HTML video generator`

Do not rename the repository after publishing unless needed. GitHub search, stars, backlinks, and clone URLs work better when the name stays stable.
